Introduction

SYS/BIOS 6.45.00.20 is a GA of SYS/BIOS 6.45. CCS users must use CCSv6.0.0 or higher.

Please read the SYS/BIOS User’s Guide for help installing and setting up your environment to use SYS/BIOS 6.45.00.20.

What’s New

SYS/BIOS 6.45.00.20 GA Release (this release).

  • Added a MacOS installer
  • Removed DSP/BIOS legacy support package
  • Removed support for C64x+ (C64P) and Tesla (C64T) targets
  • Removed support for TI A8F target (TI A8Fnv and GCC A8F are still supported and should be used instead of TI A8F target)
  • Removed support for TI MSP430 small-code small-data model target (MSP430X_small). MSP430 large-code restricted-data model target (MSP430X) is still supported and should be used instead of MSP430X_small target.
  • Removed support for legacy Stellaris LM3xx/LM4xx devices
  • ROV can now detect and work with constructed objects that are embedded within other objects (Requires XDC Tools 3.32.00 or newer).
  • Added Cortex-A15 support for more Keystone2 family devices (K2E, K2L and K2K)
  • Added GNU support for dual-core Cortex-M3/M4 present in TI’s multi-core family of devices such as OMAP4, OMAP5, AM57xx and J6.
  • Added a Seconds delegate module for MSP432 (uses RTC)
  • Added a Memory Protection (MPU) module for Cortex-M devices that support one.
  • Added support for POSIX sleep() and usleep() functions
  • Miscellaneous bug fixes. See the bug list for more info.

Upgrade and Compatibility Information

SYS/BIOS 6.40, 6.41 and 6.42

  • All modules except GIO and DMTimer are binary compatible for all targets.
  • Applications using GIO or DMTimer modules must recompile all source files that use these modules.

SYS/BIOS 6.35 and 6.37

  • All modules except GIO and DMTimer are binary compatible for all targets except MSP430 and C2000.
  • MSP430 and C2000 users must recompile all source files that use the Semaphore, Mailbox, or Event modules. The timeout parameter was changed from UInt to UInt32 which are different sizes.
  • Applications using GIO or DMTimer modules must recompile all source files that use these modules.

SYS/BIOS 6.34 and older

  • Source compatible. All APIs are supported but all source files must be recompiled.

Versioning

All releases have 4 digits (M.mm.pp.bb). This includes GA and pre-releases (engineering, alpha/EA, beta, etc.). Pre-releases are denoted with a suffix (e.g. 6.30.00.00-eng or 6.30.00.10-beta).

This product’s version follows a version format, M.mm.pp.bb, where M is a single digit Major number, mm is 2 digit minor number, pp is a 2 digit patch number, and bb is an unrestricted set of digits used as an incrementing build counter.

To support multiple side-by-side installations of the product, the product version is encoded in the top level directory (e.g. bios_6_30_00_00).

Subsequent releases of patch upgrades will be identified by the patch number, ex. SYS/BIOS 6.30.01.15 with directory bios_6_30_01_15. Typically, these patches only include critical bug fixes.

Please note that version numbers and compatibility keys are NOT the same. For an explanation of compatibility keys, please refer to the Upgrade and Compatibility Information section.
