32 /*!***************************************************************************
33  * @file SDSPI.h
34  *
35  * @brief SD driver implementation built on the TI SPI driver.
36  *
37  * The SDSPI header file should be included in an application as follows:
38  * @code
39  * #include <ti/drivers/SD.h>
40  * #include <ti/drivers/sd/SDSPI.h>
41  * @endcode
42  *
43  * Refer to @ref SD.h for a complete description of APIs & example of use.
44  *
45  * This SD driver implementation can be used to communicate with SD cards
46  * via a SPI peripheral. This driver leverages the TI SPI driver to transfer
47  * data to/from the host processor to the SD card. The SD card chip select
48  * is also handled by this driver via the TI GPIO driver. Both the SPI
49  * driver instance & the GPIO pin (used as chip select) must be specified
50  * in the SDSPI hardware attributes.
51  *
52  * Note: This driver requires that the 'defaultTxBufValue' field in the SPI
53  * driver hardware attributes be set to '0xFF'.
54  *
55  * ## Data location & alignment #
56  *
57  * This driver relies on the TI SPI driver to configure the SPI peripheral &
58  * perform data transfers. This means that data to be transferred must comply
59  * with rules & restrictions set SPI driver (memory alignment & DMA
60  * accessibility requirements). Refer to @ref SPI.h & the device specific
61  * SPI implementation header files for details.
62  *
63  * <hr>
64  */
